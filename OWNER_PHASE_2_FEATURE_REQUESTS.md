\# CIBI Platform — Phase 2 Owner Feature Requests



Following the owners’ review, the following advanced features have been requested for the next development phase.



\## 1. Super Admin, Rector, Lecturer and Role-Based Access



The platform should support a proper role hierarchy.



\### Required roles



\* Super Admin

\* Rector / Senior Admin

\* Lecturer

\* Student



\### Super Admin Access



The Super Admin should have full control over the entire platform, including:



\* Creating admin accounts

\* Creating lecturer accounts

\* Assigning courses to lecturers

\* Giving selected senior users, such as the Rector, wider access

\* Managing all courses, students, payments, certificates, settings and reports



There can be more than one Super Admin if the main Super Admin decides to grant that level of access to trusted persons.



\### Lecturer Access



Lecturers should only have access to courses assigned to them by the Super Admin.



Lecturers should be able to:



\* View their assigned courses

\* Manage lessons for their courses if permitted

\* Manage assignments and quizzes for their courses

\* View only students enrolled in their courses

\* Reply to course discussions

\* Manage live class chat settings for their own class



Lecturers should not have access to all courses unless the Super Admin grants them extended permissions.



\## 2. Dollar Pricing and Currency Conversion



All course prices should be displayed primarily in US Dollars.



The platform should include a currency converter or calculator that allows students to estimate the course fee in their local currency.



The converter should support:



\* USD as the base price

\* Student-selected local currency

\* Approximate converted value

\* Admin-controlled exchange rate or live exchange rate integration where possible



\## 3. Lecturer Student Grouping



Lecturers should be able to divide students into groups for class tasks, discussions and assignments.



Example:



If a lecturer has 300 students, they should be able to group them into smaller groups such as 10 students per group.



The grouping system should allow:



\* Automatic group creation

\* Manual group assignment

\* Group names

\* Group-based tasks or assignments

\* Lecturer view of each group and members



\## 4. Video Assignment Submission



Students should be able to submit video assignments using a link.



Supported video submission formats may include:



\* YouTube unlisted link

\* Vimeo link

\* Google Drive link

\* Loom link

\* Other approved external video links



Lecturers/admin should be able to view the submitted video link, grade the assignment and give feedback.



\## 5. Multilingual Subtitles and Translation



The learning platform should support subtitles for video lessons.



Students should be able to view subtitles in their preferred language where available.



Possible options:



\* Admin uploads subtitle files for each video

\* AI-generated subtitles/translations in a future advanced phase

\* Multiple subtitle language options per lesson



\## 6. Live Class Recording and Replay



Any live class should remain available on the platform after the live session ends so students can rewatch it later.



The replay should be attached to the relevant course or live class record.



Students should be able to:



\* Watch the live class while it is active

\* Rewatch the recorded class later

\* Access past live classes from their student portal



\## 7. Hidden Live Streaming Provider



The live stream should play directly inside the CIBI student portal.



Students should not need to know the external platform used for streaming.



The system should embed the live stream in a way that feels fully native to CIBI.



\## 8. Lecturer-Controlled Live Chat



Lecturers should be able to turn live class chat on or off from their dashboard.



When chat is off:



\* Students can watch the live class

\* Students cannot send chat messages

\* The lecturer can reopen chat when needed



\## 9. Voice Response During Live Classes



Lecturers should be able to ask questions during live classes and allow selected students to respond by voice.



This requires a real-time audio/live classroom system.



Possible options include:



\* Push-to-talk student response

\* Lecturer grants microphone access to a student

\* Lecturer mutes/unmutes student participation

\* Voice interaction inside the live class room



This feature may require integration with a real-time video/audio provider and should be treated as an advanced production feature.



\## Recommended Development Phases



\### Phase 2A — Role and Lecturer Control



\* Super Admin role

\* Rector/Admin permission control

\* Lecturer accounts

\* Lecturer-course assignment

\* Lecturer dashboard

\* Lecturer-only course access



\### Phase 2B — Pricing and Group Management



\* USD pricing

\* Currency converter

\* Lecturer student grouping

\* Group assignments/tasks



\### Phase 2C — Advanced Learning Features



\* Video assignment links

\* Subtitles

\* Live replay library

\* Hidden live streaming embed

\* Lecturer chat control



\### Phase 2D — Real-Time Voice Classroom



\* Student microphone participation

\* Lecturer mute/unmute control

\* Live audio response system

\* Real-time classroom provider integration



\## Important Technical Note



Some of these features, especially live voice response, replay handling, multilingual subtitles and scalable live streaming, may require third-party services and paid infrastructure. They should be planned carefully before public launch at scale.
